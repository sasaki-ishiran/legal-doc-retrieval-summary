# RAG + CoT 提示词消融实验说明

## 1. 实验必要性

在长篇法律文书智能分析任务中，直接将全文截断后输入大模型会导致两个问题：第一，文书后半部分的“本院认为”“判决如下”等关键裁判信息可能被截断；第二，大模型在证据不足时可能根据常见法律文书模板补全内容，产生事实性幻觉。因此，本系统先引入 RAG 分片机制，从全文中筛选证据片段，再将片段输入大模型。

在此基础上，引入 CoT（Chain-of-Thought，链式思维）或结构化推理提示词具有实验价值。法律文书分析并不是单纯摘要任务，而是需要先定位事实、争议、法院认定、法律依据和裁判结果，再完成摘要与实体抽取。加入“内部逐步推理 + 证据核验”提示，有助于约束模型先检查证据，再输出结论，从而提升关键要素抽取完整性和降低幻觉率。

需要注意的是，论文和系统中不建议直接输出完整推理链。原因是法律场景对结论可验证性和输出简洁性要求较高，直接展示大模型推理过程可能引入冗余甚至错误解释。因此，本项目采用“隐式 CoT”策略：提示模型内部逐步分析和自检，但最终只输出 JSON 结果。

## 2. 对比方案设计

建议设置如下消融实验：

| 实验编号 | 输入策略 | 提示词策略 | 目的 |
| --- | --- | --- | --- |
| A | 直接截断 | 基础提示词 | 作为原始基线，验证长文书截断问题 |
| B | RAG 分片 | 基础证据约束提示词 | 验证 RAG 对关键内容召回的提升 |
| C | RAG 分片 | RAG + 隐式 CoT/证据核验提示词 | 验证结构化推理提示对摘要与抽取质量的提升 |

当前代码支持通过环境变量切换提示词策略：

```bash
# RAG + CoT，默认策略
set LEGAL_IR_LLM_PROMPT_STRATEGY=rag_cot

# RAG + 基础证据约束提示词
set LEGAL_IR_LLM_PROMPT_STRATEGY=basic
```

## 3. 评价指标

可以从以下角度评价：

1. 摘要完整性：摘要是否覆盖案情事实、争议焦点、法院认定和裁判结果。
2. 实体抽取准确率：案由、法院、当事人、法律条款、判决结果是否正确。
3. 幻觉率：输出中是否出现证据片段没有支持的信息。
4. JSON 合法率：模型输出是否为合法 JSON，能否被系统解析。
5. 长文书后置信息命中率：摘要或实体结果是否包含“本院认为”“判决如下”等后置关键段落信息。

如有人工标注样本，可以计算 Precision、Recall、F1；如样本规模较小，也可以采用人工评分表，按 1-5 分评价完整性、准确性和可用性。

## 4. 论文写作表述示例

为进一步降低大模型在法律文书分析中的幻觉风险，本文在 RAG 分片证据输入的基础上设计了提示词消融实验。基础提示词仅要求模型依据证据片段完成摘要和实体抽取；改进提示词引入隐式链式思维机制，要求模型在内部先逐段阅读证据，识别案件事实、争议焦点、法院认定、法律依据和裁判结果，再对每个字段进行证据核验，最后输出结构化 JSON。考虑到法律场景对输出可控性和可解释边界的要求，系统不直接输出模型推理链，而是将 CoT 作为内部分析约束使用。

实验对比了“直接截断 + 基础提示词”“RAG 分片 + 基础提示词”和“RAG 分片 + 隐式 CoT 提示词”三种方案。实验结果表明，直接截断方案容易遗漏位于文书后半部分的裁判理由和判决结果；RAG 分片能够显著提升关键片段覆盖率；在 RAG 基础上加入隐式 CoT 与证据核验提示后，模型输出在摘要完整性、字段一致性和幻觉控制方面进一步改善，说明该提示词策略适合法律文书智能分析任务。

## 5. 已在系统中实现的提示词策略

当前系统在 `LeCaDR/llm_utils.py` 中实现了两类提示词：

1. 基础证据约束提示：要求模型只能依据证据片段生成结论。
2. 隐式 CoT 提示：要求模型内部进行逐步阅读、字段证据核验和 JSON 自检，但最终不输出推理链。

默认策略为 `rag_cot`，可通过 `LEGAL_IR_LLM_PROMPT_STRATEGY` 环境变量切换。这样既能满足系统实际效果优化，也能支撑论文中的提示词消融实验章节。
